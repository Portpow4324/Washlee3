🔧 KEY FEATURES
═══════════════════════════════════════════════════════════════════════════

✅ Returning User Flow
   1. User logs in → Check phone_verified status
   2. If false → Redirect to phone-verification page
   3. User enters phone → Code sent via email
   4. User enters code → Verify against database
   5. Success → Set phone_verified = TRUE
   6. Redirect → User dashboard

✅ Dev Mode Testing
   • URL: http://localhost:3000/auth/phone-verification?dev=true
   • Display: Yellow box with test 6-digit code
   • Button: "Auto-fill" to populate input
   • Logs: Console shows all codes
   • Benefit: No external service needed

✅ Phone Persistence
   • Stored: In users table (master record)
   • Synced: To customers & employees tables
   • Prevents: Duplicate registration
   • Loads: Auto-populated on return visits

✅ Error Handling
   • Invalid format → User-friendly message
   • Duplicate phone → Clear explanation
   • Expired code → Suggest "Resend"
   • Network error → Graceful fallback
